В системе доступен предопределённый промпт, который одинаково подходит для [формирования резюме диалога к звонку
или чату](dialog-analyze.md).
При необходимости вы можете создавать и тестировать собственные промпты.

## Создание промпта

{.miko-man}
1. В панели разделов выберите [!badge Контакт-центр] :icon-chevron-right: [!badge Сервисы AI] :icon-chevron-right: [!badge Разработка промптов].
2. Добавьте новый промпт, нажав кнопку [!badge Создать].
3. Заполните следующие параметры:

   Модель
   : Выберите модель AI, которая будет использоваться для обработки текста:
      - YandexGPT Lite - подходит для быстрых ответов и типовых задач, например для краткого резюме диалога.
      - YandexGPT Pro - подходит для более сложных задач, где требуется более детальный анализ содержания.
   
   Промпт
   : Укажите инструкцию для модели с описанием ожидаемого результата и формата ответа.
    
   !!!tip Совет
   При передаче транскрибации звонка реплики оператора обозначаются как "О", а реплики клиента — как "К".
   Рекомендуется учитывать это в тексте инструкции, если промпт должен различать участников диалога.
   Обозначения сокращены для уменьшения количества затрачиваемых токенов.
   !!!

   Температура
   : Определяет вариативность ответа модели.
   Низкие значения делают ответ более предсказуемым и стабильным, высокие — более разнообразным.

   Максимальное количество токенов
   : Ограничивает максимальный объём ответа, который может сгенерировать модель.

## Рекомендации

- Для краткого резюме диалогов обычно достаточно модели **YandexGPT Lite**.
- Если требуется более подробный анализ, выделение намерений клиента или извлечение договорённостей,
используйте **YandexGPT Pro**.
- Для формализованных и повторяемых ответов рекомендуется задавать невысокую температуру.
- Если ответ получается слишком длинным или слишком коротким, скорректируйте параметр максимального количества токенов.
- Перед использованием промпта в рабочем процессе рекомендуется протестировать его на нескольких диалогах разного типа.

## Готовые промпты

==- Краткое резюме
```
Проанализируй диалог между оператором и клиентом и подготовь краткое резюме в строго заданном формате Markdown.

Обозначения участников:
- О — оператор
- К — клиент

Требования:
- Используй только сведения из диалога.
- Не добавляй предположения и интерпретации, которых нет в тексте.
- Пиши кратко и по существу.
- Ответ дай на русском языке.
- Сохрани структуру и заголовки без изменений.

Формат ответа:

## Тема обращения
Кратко укажи, по какому вопросу обратился клиент.

## Суть диалога
Опиши в 2–4 предложениях, что обсуждалось в разговоре.

## Результат
Укажи, к какому результату пришли стороны по итогам диалога.
Если результат не достигнут, так и напиши.

## Важные детали
- ...
- ...

Правила заполнения:
- В разделе «Важные детали» указывай только значимые факты: даты, суммы, номера заказов, адреса, контакты, договорённости, ограничения.
- Если важных деталей нет, укажи:
- Нет
```
==- Выделение задач для сотрудника
```
Проанализируй диалог между оператором и клиентом и выдели задачи для сотрудника по итогам разговора.

Обозначения участников:
- О — оператор
- К — клиент

Требования:
- Выделяй только явные задачи, которые следуют из содержания диалога.
- Не придумывай действия, если они прямо не вытекают из разговора.
- Ответ дай на русском языке.
- Используй строго заданный формат Markdown.
- Если задач нет, явно укажи это в соответствующем разделе.

Формат ответа:

## Наличие задач
Да / Нет

## Перечень задач
1. **Задача:** ...
    - **Срок:** ...
    - **Связанные данные:** ...
    - **Основание:** ...

2. **Задача:** ...
    - **Срок:** ...
    - **Связанные данные:** ...
    - **Основание:** ...

## Комментарий
Кратко укажи, требуется ли дальнейшее действие со стороны сотрудника.

Правила заполнения:
- В поле «Срок» указывай срок только если он явно упоминается в диалоге. Иначе пиши: не указан.
- В поле «Связанные данные» указывай номер заказа, телефон, адрес, сумму, дату или иные существенные данные, если они есть. Иначе пиши: нет.
- В поле «Основание» кратко укажи, из какой договорённости или просьбы клиента следует задача.
- Если задач нет, верни такой результат:

## Наличие задач
Нет

## Перечень задач
Нет

## Комментарий
По итогам диалога явные задачи для сотрудника не выявлены.
```
==- Оценка настроения клиента
```
Проанализируй реплики клиента в диалоге и оцени его настроение.

Обозначения участников:
- О — оператор
- К — клиент

Требования:
- Оценивай настроение только по содержанию реплик клиента.
- Не делай выводов, которые нельзя подтвердить текстом.
- Используй только следующие значения оценки:
    - позитивное
    - нейтральное
    - негативное
- Если настроение менялось по ходу разговора, отрази это в ответе.
- Ответ дай на русском языке.
- Используй строго заданный формат Markdown.

Формат ответа:

## Оценка настроения
позитивное / нейтральное / негативное

## Динамика
Укажи, изменялось ли настроение клиента в ходе диалога.
Если не изменялось, напиши: Не выявлена.

## Основание оценки
- ...
- ...

## Комментарий
В 1–3 предложениях кратко объясни вывод.

Правила заполнения:
- В разделе «Основание оценки» перечисляй конкретные признаки из диалога: жалобы, благодарность, недовольство, спокойные уточнения, раздражение, согласие, отказ и т. п.
- Если данных для уверенной оценки недостаточно, выбери наиболее нейтральную оценку и укажи это в комментарии.
```
==- Извлечение договорённостей
```
Проанализируй диалог между оператором и клиентом и выдели договорённости, достигнутые в ходе разговора.

Обозначения участников:
- О — оператор
- К — клиент

Требования:
- Учитывай только те договорённости, которые прямо сформулированы в диалоге.
- Не добавляй предположения и не интерпретируй неясные формулировки как достигнутую договорённость.
- Если договорённостей нет, явно укажи это.
- Ответ дай на русском языке.
- Используй строго заданный формат Markdown.

Формат ответа:

## Наличие договорённостей
Да / Нет

## Перечень договорённостей
1. **Договорённость:** ...
   - **Инициатор:** клиент / оператор / обе стороны
   - **Срок или дата:** ...
   - **Связанные данные:** ...
   - **Основание:** ...

2. **Договорённость:** ...
   - **Инициатор:** клиент / оператор / обе стороны
   - **Срок или дата:** ...
   - **Связанные данные:** ...
   - **Основание:** ...

## Комментарий
Кратко укажи, насколько договорённости определённые и требуют ли дальнейшего контроля.

Правила заполнения:
- В поле «Срок или дата» указывай только явно названные сроки, даты или интервалы. Иначе пиши: не указано.
- В поле «Связанные данные» указывай номер заказа, телефон, адрес, сумму, способ связи или другие значимые данные, если они есть. Иначе пиши: нет.
- В поле «Основание» кратко укажи, какая реплика или договорённость подтверждает вывод.
- Если договорённостей нет, верни такой результат:

## Наличие договорённостей
Нет

## Перечень договорённостей
Нет

## Комментарий
По содержанию диалога явные договорённости не зафиксированы.
```
==- Определение темы обращения
```
Проанализируй диалог между оператором и клиентом и определи тему обращения клиента.

Обозначения участников:
- О — оператор
- К — клиент

Требования:
- Определи основную тему обращения по содержанию диалога.
- При наличии второстепенных тем укажи их отдельно.
- Формулируй тему кратко, без лишних подробностей.
- Не добавляй сведения, которых нет в диалоге.
- Ответ дай на русском языке.
- Используй строго заданный формат Markdown.

Формат ответа:

## Основная тема
...

## Дополнительные темы
- ...
- ...

## Основание
В 1–3 предложениях укажи, по каким фрагментам диалога определена тема обращения.

## Ключевые сущности
- ...
- ...

Правила заполнения:
- В разделе «Основная тема» укажи одну главную тему обращения.
- В разделе «Дополнительные темы» перечисляй только реально присутствующие сопутствующие вопросы. Если их нет, укажи:
- Нет
- В разделе «Ключевые сущности» перечисляй важные объекты из диалога: товар, услуга, заказ, доставка, оплата, возврат, визит, адрес, дата и т. п.
- Если тема обращения определена неуверенно, выбери наиболее вероятную формулировку и укажи это в разделе «Основание».
```
==- Контроль качества ответа оператора
```
Проанализируй диалог между оператором и клиентом и оцени качество работы оператора.

Обозначения участников:
- О — оператор
- К — клиент

Требования:
- Оценивай только действия и ответы оператора по тексту диалога.
- Не делай выводов, которые нельзя подтвердить содержанием разговора.
- Оцени, насколько оператор:
  - понял запрос клиента;
  - дал корректный и понятный ответ;
  - соблюдал вежливый и деловой стиль общения;
  - довёл диалог до результата или следующего понятного шага.
- Ответ дай на русском языке.
- Используй строго заданный формат Markdown.

Формат ответа:

## Общая оценка
хорошо / удовлетворительно / неудовлетворительно

## Понимание запроса
...
    
## Качество ответа
...

## Стиль общения
...

## Результативность
...

## Замечания
- ...
- ...

## Итоговый комментарий
В 2–4 предложениях кратко опиши сильные и слабые стороны ответа оператора.

Правила заполнения:
- В разделах «Понимание запроса», «Качество ответа», «Стиль общения» и «Результативность» давай краткую оценку с опорой на текст диалога.
- В разделе «Замечания» перечисляй только конкретные недочёты или спорные моменты. Если замечаний нет, укажи:
- Нет
- Оценку «хорошо» ставь, если оператор правильно понял запрос, вёл диалог корректно и дал клиенту понятный результат или следующий шаг.
- Оценку «удовлетворительно» ставь, если в ответе есть недочёты, но в целом оператор помог клиенту.
- Оценку «неудовлетворительно» ставь, если оператор не понял запрос, дал неполный или некорректный ответ либо не помог продвинуться к решению вопроса.
```
==-
